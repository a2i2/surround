Command-line Interface¶

The following is a list of the sub-commands contained in Surround’s CLI tool.

surround¶

The Surround Command Line Interface

usage: surround [-h] {init,run,lint,remote,add,pull,push,list} ...

init¶

Initialize a new Surround project.

usage: surround init [-h] [-p PROJECT_NAME] [-d DESCRIPTION] [path]

Positional Arguments¶

path

Path for creating a Surround project

Default: “./”

Named Arguments¶

-p, --project-name

Name of the project

-d, --description

A description for the project

run¶

Run a Surround project task, witout an argument all tasks will be shown.

Tasks are defined in the dodo.py file of the current project. The default tasks that come with every project are:

  • build - Build a Docker image for your Surround project.

  • dev - Run the Docker image with the current source code (via drive mount).

  • prod - Build and run the Docker image (no drive mounting).

usage: surround run [-h] [-w] [task]

Positional Arguments¶

task

Task defined in a Surround project dodo.py file.

Named Arguments¶

-w, --web

Name of the class inherited from Wrapper

Default: False

lint¶

Run the Surround Linter on the current project.

For more information on what this does, see Linter.

usage: surround lint [-h] [-l | path]

Positional Arguments¶

path

Path for running the Surround linter

Default: ./

Named Arguments¶

-l, --list

List all Surround checkers

Default: False
